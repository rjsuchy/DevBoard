Question-1: What are the different ways to select an element in the DOM?

We can use different methods to select elements in HTML. These methods include getElementById for selecting an element by its ID, getElementsByClassName for selecting elements by their class name, getElementsByTagName for selecting elements by their tag name, and querySelector or querySelectorAll for selecting elements using CSS selectors.

Question-2: What is the difference between innerHTML, innerText, and textContent?

innerHTML lets us see and change everything inside an element, including the HTML tags and how it looks. It’s like viewing the actual code for that section of the webpage.
innerText shows only the text that is visible to users on the screen. It ignores any hidden text and takes into account the styling, making it great for getting just the text that people can read.
textContent gives us all the text inside an element, even if some of it is hidden, but it doesn’t show any HTML tags or formatting. It’s like reading plain text, including parts that users can’t see.

Question-3: How do you add an event listener in JavaScript?

We can use addEventListener in JavaScript to respond to events. For instance, we can select a button with document.querySelector('button'). After selecting the button, we can add an event listener like this: button.addEventListener('click', () => alert('Clicked!'));. This means that when we click the button, the code inside the function will run, and an alert will pop up saying "Clicked!". This is an easy way for us to make our webpage interactive and engage users!

Question-4: What is the purpose of the 'this' keyword in JavaScript?

In JavaScript, "this" refers to the object that is currently running the code. The meaning of "this" can change depending on how we call a function. It helps us understand which object the methods and properties belong to. So, "this" is important for keeping track of the context in our code!

Question-5: How do you create a promise in JavaScript?

In JavaScript, we use the Promise constructor to handle async tasks. We pass a function with resolve and reject parameters. If the task succeeds, we call resolve(value); if it fails, we use reject(error). This makes async code cleaner and easier to manage!
